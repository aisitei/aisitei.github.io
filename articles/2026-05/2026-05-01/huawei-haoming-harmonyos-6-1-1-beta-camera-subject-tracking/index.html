화웨이 하오밍 HarmonyOS 6.1.1 (24) 베타 1 버전 출시, 카메라 키트로 카메라 컨트롤러에 '피사체 추적' 기능 제공

2026-05-01

HarmonyOS 6.1.1 Beta1 개발자 업데이트 주요 내용

IT之家에 따르면, Huawei 개발자 공식 웹사이트를 통해 HarmonyOS 6.1.1 (24) Beta1 버전이 공개되었습니다. 이 버전은 이전 6.1.0 (23) 버전을 기반으로 하며, 개발 역량이 대폭 강화된 것이 특징입니다.

핵심 프레임워크 기능 강화

주요 모듈별로 다음과 같은 성능 및 기능을 개선했습니다:

  • Ability Kit: AbilityStage의 컨텍스트(Context) 능력을 향상시켜 동적 리소스 로딩을 지원합니다.
  • ArkUI: 평행 시야 상태 획득, 사용자 정의 컴포넌트의 Ability 간 마이그레이션, 동적 레이아웃 컨테이너 등 기능을 강화했으며, 여러 컴포넌트에 대한 C API가 추가되었습니다.
  • ArkTS: 가상 머신 측정 능력을 향상시키고 taskpool 작업 타임아웃 설정을 지원합니다.
  • ArkWeb: 다운로드 작업 콜백 기능을 개선하고 URL 화이트리스트 및 보안 제어 인터페이스를 추가했습니다.
  • Camera Kit: 지연 미리보기 출력(Delayed Preview Output)과 인물 추적(Auto Framing) 기능을 새로 추가했습니다.
  • Audio Kit: 외부 장치 연결을 위한 MIDI C API 지원이 추가되었습니다.
  • FAST Kit: 동시 해시 테이블, 벡터 연산 및 필터 기능이 추가되었습니다.
  • Performance Analysis Kit: 리소스 수집 및 충돌 로그 분석 능력이 강화되었습니다.

또한, Content Embed Kit(콘텐츠 임베드 서비스)와 Enterprise Threat Protection Kit(기업 위협 방지 서비스) 등 새로운 키트가 추가되었습니다.

개발 환경 (DevEco Studio) 개선 사항

개발 도구인 DevEco Studio의 기능도 크게 향상되었습니다:

  • API 지원: API 24 엔지니어링을 지원합니다.
  • Hot Reload 강화: C++ 코드와 리소스 파일 수정까지 지원하는 Hot Reload 기능을 개선했습니다.
  • 디버깅 효율성 증대: Binder 통신 정보, 메인 스레드 작업 큐 및 샘플링 스택 데이터를 포함하여 더 많은 AppFreeze 로그를 분석할 수 있게 되어 개발자가 문제점을 찾는 효율성이 높아졌습니다.
  • 메모리 누수 진단: UI 인터페이스 각 컴포넌트의 메모리 할당 상황을 분석하는 ComMemory 템플릿이 추가되어, UI 컴포넌트의 메모리 누수를 찾는 데 도움을 줍니다.

주요 API 및 기능 상세 업데이트

HarmonyOS 6.1.1 (24) Beta1 버전에서 새롭게 지원되거나 개선된 세부 기능들은 다음과 같습니다.

시스템 및 앱 개발

  • Ability 관리: Ability의 마지막 종료 정보 필드에 '종료 원인'을 가져오는 것이 추가되었습니다.
  • AbilityStage 컨텍스트: `launchElement` 필드가 추가되어, AbilityStage가 onCreate를 호출할 때 현재 로딩 중인 Ability 정보를 앱에 전달함으로써 동적 리소스 로딩이 가능해졌습니다.
  • 위젯 및 데이터베이스: 애플리케이션 내 바로가기 검색 및 삭제 기능 지원이 추가되었으며, 관계형 데이터베이스(RdbStore)를 생성하거나 기존 DB를 여는 동기화 방법도 새로 추가되었습니다.
  • HDR/AI 지원: 컴포넌트 내용에 HDR(High Dynamic Range Imaging) 밝기 효과를 적용하는 것이 가능해졌고, 비디오의 AIHDR 포맷을 지원합니다.

가상 머신 및 성능 분석

  • 메모리 정보 수집: 모든 가상 머신 스레드의 힙 메모리 정보(스레드 ID, 이름, 힙 유형, 객체 크기 포함)를 가져오는 것이 가능해졌습니다.
  • 메모리 누수 방지: `enableLocalHandleDetection` 인터페이스가 추가되어 EventHandler 및 libuv 메커니즘의 작업이 스코프 내에서 실행되도록 보장하여 메모리 누수를 예방합니다.
  • 타임아웃 설정: taskpool의 execute 메서드를 강화하여, 작업이나 작업 그룹에 타임아웃 시간을 지정할 수 있게 되었으며, 초과 시 오류 메시지를 발생시킵니다.

UI/UX 및 컴포넌트

  • 평행 시야(Parallel View): 앱이 평행 시야 기능을 능동적으로 사용할 때, 새로운 API 인터페이스를 통해 현재 분할 화면 상태를 런타임에 가져와 비즈니스 로직이나 UX 애니메이션을 조정하기 편리해졌습니다.
  • 컴포넌트 유연성: 사용자 정의 컴포넌트가 Ability 간 마이그레이션을 지원하게 되었습니다.
  • C API 확장: 여러 컴포넌트에 대한 C API(예: OH_ArkUI_DecorationStyleOptions 등)가 추가되었으며, 속성 문자열을 가져오는 C API도 추가되었습니다.
  • 사용자 경험 개선: 텍스트 컴포넌트에서 좌표를 기반으로 가장 가까운 문자의 위치 정보를 얻는 것이 가능해졌습니다. 또한, 드래그앤드롭 비동기 알림 인터페이스가 추가되어 복사/잘라내기 처리 방식과 애니메이션 실행 여부를 지정할 수 있습니다.
  • 소프트 키보드: `onNeedSoftkeyboard` 콜백이 새로 추가되어 개발자가 포커스 이동 후에도 소프트 키보드가 닫히지 않도록 설정할 수 있게 되었습니다.

미디어 및 카메라

  • 오디오 기능: 오디오 녹음 및 렌더링에 독립적인 오디오 세션 전략과 동작 매개변수를 설정하는 것이 가능해졌습니다.
  • MIDI 지원: OH_MIDI C API가 추가되어, 앱이 USB 또는 블루투스 BLE를 통해 외부 MIDI 장치(키보드, 전자 피아노 등)와 연결하여 MIDI 메시지를 송수신하고 장치를 열거/핫 플러그 감지할 수 있습니다.
  • 코덱 및 보안: Cinepak 미디어 포맷의 코덱 지원이 추가되었으며, DRM 보호 콘텐츠를 처리할 때 안전한 링크를 지원하는 '안전 디코더'를 사용하여 특정 MIME 유형을 필터링할 수 있게 되었습니다.
  • 카메라 기능: 카메라 컨트롤러에 인물 추적(AUTO_FRAMING) 능력이 제공되어, 사람이 화면 중앙에 항상 위치하도록 자동으로 자르고 확대합니다.

지도 및 보안

  • 지리 정보: 지도 앱에서 오프라인 지도 관리 페이지와 지점 상세 정보를 불러오는 것이 가능해졌습니다. 또한, 마커(Marker)나 POI를 길게 누르는 이벤트 리스너 기능이 추가되었습니다.
  • Kiosk 모드: Kiosk 모드에서 스와이프 업 정지 제스처로 최근 작업 표시줄에 진입하거나, 가장자리 내부 스와이프 정지 제스처로 사이드 DOCK 바에 진입하는 것이 가능해졌습니다.
  • 보안 및 권한: 앱 간 문서 임베딩 및 협업 편집을 위한 Content Embed Kit가 제공되며, 코드 서명 정보 조회 기능이 추가되었습니다. 또한, 시스템 보안 관련 이벤트(프로세스 권한 상승, 프로세스 비정상 디버깅 등)를 처리하는 기능들이 대거 추가되었습니다.

디버깅 및 테스트 도구

  • 리소스 수집 강화: HiDebug에서 애플리케이션 프로세스의 물리 메모리 사용 정보를 가져오는 것이 가능해졌으며, 스냅샷을 스레드 단위가 아닌 프로세스 단위로 전송할 수 있게 되었습니다.
  • 트레이싱 개선: HiDebug는 커널 정보 등을 포함한 트레이스(Trace) 수집 요청 인터페이스를 제공합니다.

총평

HarmonyOS 6.1.1 Beta1은 개발자 생태계의 깊이와 폭을 동시에 확장하는 대규모 업데이트입니다. 특히 Ability Kit, ArkUI 등 핵심 프레임워크 전반에 걸쳐 동적 로딩, 메모리 관리, 사용자 경험(UX) 관련 기능들이 강화되어 앱 개발의 유연성이 크게 높아졌습니다. 또한, MIDI 지원, AIHDR 포맷 도입, 그리고 강력해진 디버깅 및 보안 기능을 통해 플래그십 기기 수준의 전문적인 애플리케이션 구현이 가능해지면서, HarmonyOS 생태계가 더욱 성숙하고 견고한 플랫폼으로 진화했음을 보여줍니다.
