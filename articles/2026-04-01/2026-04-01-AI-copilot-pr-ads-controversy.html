Copilot, 개발자 PR에 무단 광고 삽입 — GitHub 긴급 철회

📅 2026년 3월 31일
🔗 IT之家

개발자들의 강력한 반발로 인해 GitHub는 Copilot의 행동에 대해 긴급 조정을 단행했습니다. 그동안 Copilot이 사용자들이 작성한 Pull Request(PR)에 몰래 "팁(tips)"이라는 명목의 광고 콘텐츠를 삽입해온 것으로 드러났습니다.

광고 삽입 논란의 발단

호주 개발자 Zach Manson은 동료가 Copilot에게 자신의 PR에서 철자 오류를 수정하도록 요청한 후, Copilot이 추가한 메시지를 발견했습니다. 해당 메시지는 Raycast 도구와 설치 링크를 홍보하는 광고성 콘텐츠였습니다. Manson은 이러한 행동이 "정말 짜증난다"라고 표현했으며, Copilot이 자신이 작성하지 않은 PR의 설명과 댓글을 수정할 권한을 가지고 있다는 사실에 충격을 받았습니다.

광고 범위의 광범위함

검색 결과에 따르면 11,400개 이상의 PR에서 동일한 광고 내용이 발견되었습니다. 이는 문제의 심각성을 보여주는 명백한 증거입니다.

GitHub의 공식 입장

GitHub 개발자 관계 부사장 Martin Woodward는 다음과 같이 입장을 밝혔습니다. "Copilot이 자신이 생성하지 않은 PR을 수정하도록 하는 것은 새로운 시도였으나, 현재로서는 그 효과가 좋지 못한 것으로 판단됩니다."

Woodward는 강조하며 말했습니다: "GitHub은 현재 GitHub 내에 광고를 포함하지 않고 있으며, 향후 광고를 추가할 계획도 없습니다."

Copilot 개발팀의 해명

GitHub Copilot의 수석 제품 관리자 Tim Rogers는 이 조치의 초기 의도는 개발자들에게 에이전트를 사용하는 새로운 방식에 대해 알려주는 것이었다고 설명했습니다. 그러나 이는 잘못된 판단이었다고 인정했습니다.

GitHub는 현재 이러한 팁들을 비활성화했으며, 이와 같은 상황이 재발하지 않을 것을 약속했습니다.
