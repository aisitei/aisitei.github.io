OpenClaw 3.22

OpenClaw 3.22 최대 업그레이드

하위 아키텍처 대전환, 전 세계가 9일을 기다렸다

📅 2026년 3월 24일 📰 IT之家

개요

OpenClaw 3.22는 역사적인 대규모 업그레이드로, 플러그인 시스템의 완전한 개편부터 보안 강화, 모델 업그레이드, 멀티플랫폼 개선, 그리고 에이전트 엔진의 혁신까지 광범위한 변화를 가져왔습니다. 이번 업그레이드는 전 세계 개발자 커뮤니티가 9일간 기대해온 대규모 아키텍처 전환입니다.

1. 플러그인 시스템 완전 개편

OpenClaw 3.22의 가장 획기적인 변화는 플러그인 시스템의 근본적인 재구성입니다. 새로운 공개 Plugin SDK는 개발자들에게 강력한 기능을 제공하며, 구 버전 호환성 제거로 생태계의 정리와 현대화를 추진합니다.

새로운 Plugin SDK
  • 공개 Plugin SDK 출시 (openclaw/plugin-sdk/*)
  • 확장된 기능 범위
  • 개선된 개발자 경험
  • 표준화된 API 인터페이스
레거시 정책
  • 구 확장 API 완전 제거
  • 호환성 레이어 없음
  • 전환 기간 미제공
  • 모든 3rd party 플러그인 마이그레이션 필수
ClawHub 플랫폼
  • 기본 플러그인 배포 채널
  • ClawHub에서 미발견시 npm으로만 폴백
  • 중앙 집중식 관리
  • 검증된 플러그인 생태계
AI 어시스턴트 통합
  • Claude에서 플러그인 발견/설치
  • Codex 플랫폼 지원
  • Cursor IDE 통합
  • Skills 자동 매핑
마이그레이션 전략

기존 플러그인 사용자들은 즉시 새로운 Plugin SDK로 마이그레이션해야 합니다. 호환성 레이어나 전환 기간이 제공되지 않으므로, 모든 3rd party 플러그인이 신규 API를 지원해야 합니다. ClawHub은 검증된 플러그인만 호스팅하며, npm은 최후의 폴백 채널로만 기능합니다.

2. 보안 강화 (10개 이상의 패치)

OpenClaw 3.22는 10개 이상의 중요한 보안 패치를 포함하여 시스템 전반에 걸친 강화된 보안 태세를 구현합니다. 파일 시스템부터 코드 실행 환경, Unicode 처리, 웹훅 인증까지 다층 방어를 강화했습니다.

보안 패치 목록

취약점 유형 설명 대응책
Windows SMB 자격증명 유출 파일 경로 처리 취약점 file:// 및 UNC 경로를 미디어 로딩에서 차단
실행 샌드박스 우회 JVM 환경 변수 주입 MAVEN_OPTS, SBT_OPTS, GRADLE_OPTS 차단
추가 실행 우회 .NET 환경 변수 주입 GLIBC_TUNABLES, DOTNET_ADDITIONAL_DEPS 차단
유니코드 영문자 위장 0폭 문자로 승인 프롬프트 위조 한국어 필러 코드포인트 이스케이프 처리
음성 통화 웹훅 사전 인증 요청 제한 부족 1MB/30초 → 64KB/5초, IP당 동시 요청 제한
Windows SMB 자격증명 유출 차단

Windows 환경에서 UNC 경로(\\서버\공유)를 처리할 때 자격증명이 의도치 않게 노출되는 취약점을 해결했습니다. 미디어 로딩 기능에서 file:// 프로토콜과 UNC 경로를 모두 차단하여 로컬 파일시스템 접근을 제한합니다.

실행 샌드박스 강화

공격자가 환경 변수를 통해 JVM 또는 .NET 실행 환경을 조작하는 것을 방지합니다:

  • Java: MAVEN_OPTS, SBT_OPTS, GRADLE_OPTS 차단 - 빌드 프로세스 명령 주입 방지
  • .NET: GLIBC_TUNABLES, DOTNET_ADDITIONAL_DEPS 차단 - 라이브러리 로딩 조작 방지
유니코드 0폭 문자 위장 방지

가장 정교한 공격 벡터 중 하나는 0폭 문자를 이용해 사용자의 승인 프롬프트를 조작하는 것입니다. 특히 한국어 필러 코드포인트를 악용하여 실행 승인 메시지를 위장할 수 있었습니다. OpenClaw 3.22는 이러한 문자들을 자동으로 이스케이프 처리하여 화면에 표시합니다.

음성 통화 웹훅 사전 인증 제한

웹훅 사전 인증 요청의 크기와 속도를 제한합니다:

  • 이전: 1MB/30초 제한
  • 업그레이드: 64KB/5초로 강화
  • 추가: IP당 동시 사전 인증 요청 제한

이는 DDoS 및 리소스 고갈 공격을 방어합니다.

3. 모델 업그레이드

OpenClaw 3.22는 다양한 AI 모델 제공자와의 협력을 통해 최신 모델을 지원합니다. OpenAI, Anthropic, MiniMax, xAI, Z.AI, Mistral 등 주요 제공자의 업그레이드가 포함되었습니다.

모델 업그레이드 현황

제공자 변경 사항 주요 개선
OpenAI 기본 모델 → GPT-5.4 gpt-5.4-mini, gpt-5.4-nano 호환성
MiniMax M2.5 → M2.7 API/OAuth 플러그인 통합
Anthropic Vertex Google Vertex AI 추가 Claude 모델 + GCP 인증
xAI Grok 디렉토리 동기화 최신 Grok 모델 지원
Z.AI GLM 4.5/4.6 업데이트 최신 언어 모델
Mistral 가격 메타데이터 수정 정확한 모델 가격 정보
OpenAI: GPT-5.4 시대

OpenClaw의 기본 모델이 GPT-5.4로 업그레이드되었습니다. gpt-5.4-mini와 gpt-5.4-nano 변형 모델도 전방향 호환성을 갖추고 있어, 성능과 비용을 필요에 맞게 선택할 수 있습니다.

MiniMax: M2.7 출시 및 플러그인 통합

MiniMax의 최신 모델 M2.7이 지원되며, 이전에 분리되어 있던 API와 OAuth 플러그인이 단일 minimax 플러그인으로 통합되어 관리가 간편해졌습니다.

Anthropic Vertex: Google Cloud 통합

Claude 모델들이 이제 Google Vertex AI를 통해 제공됩니다. GCP 인증을 사용하여 Google Cloud 인프라에서 Claude의 강력한 성능을 활용할 수 있습니다.

xAI, Z.AI, Mistral 업데이트
  • xAI Grok: 디렉토리를 최신 상태로 동기화하여 최신 Grok 모델 지원
  • Z.AI GLM: 4.5/4.6 버전으로 업데이트된 언어 모델 제공
  • Mistral: 가격 메타데이터가 수정되어 정확한 비용 정보 제공

4. 멀티플랫폼 개선사항

OpenClaw 3.22는 Android, 제어 UI, Telegram, Feishu, 브라우저, 샌드박스 등 다양한 플랫폼에서 사용자 경험을 크게 향상시켰습니다.

🤖 Android
  • 시스템 다크모드 지원
  • 운영체제 테마 자동 동기화
  • 배터리 효율성 향상
🎨 Control UI
  • 커스터마이징 가능한 코너 반경 슬라이더
  • 개인화된 UI 디자인
  • 실시간 미리보기
📱 Telegram
  • DM 포럼 주제 자동 이름 변경 (LLM)
  • 무음 오류 응답 모드
  • 자동화된 대화 관리
🐥 Feishu
  • 구조화된 상호작용 승인 카드
  • ACP/서브 에이전트 바인딩
  • Reasoning Stream 렌더링
🌐 Browser
  • 레거시 Chrome 확장 릴레이 제거
  • userDataDir를 통한 직접 Chromium 연결
  • 성능 개선 및 단순화
🔒 Sandbox
  • 플러그 가능한 백엔드
  • OpenShell 지원
  • SSH 백엔드 통합
Android: 시스템 테마 동기화

Android 기기의 시스템 다크모드 설정을 자동으로 감지하여 OpenClaw의 테마를 동기화합니다. 이는 사용자의 기기 설정과 일관성을 유지하고 시각적 피로를 줄여줍니다.

Control UI: 커스터마이징 슬라이더

사용자가 UI 요소의 코너 반경을 자유롭게 조정할 수 있는 슬라이더를 추가했습니다. 모던한 둥근 모서리부터 날카로운 각진 디자인까지 취향에 맞게 커스터마이징 가능합니다.

Telegram: AI 기반 자동 관리

Telegram DM의 포럼 주제가 대화 내용을 분석하여 LLM이 자동으로 이름을 변경합니다. 또한 무음 오류 응답 모드로 에러를 조용히 처리하여 채팅 화면의 어수선함을 줄입니다.

Feishu: 구조화된 상호작용 및 에이전트

Feishu에서 복잡한 상호작용을 구조화된 승인 카드로 표현합니다. ACP(Assistant Cloud Platform) 및 서브 에이전트 바인딩을 지원하고, Reasoning Stream 렌더링으로 AI의 사고 과정을 실시간으로 볼 수 있습니다.

Browser: 직접 Chromium 연결

레거시 Chrome 확장 릴레이 시스템을 제거하고, userDataDir를 통해 직접 Chromium 브라우저에 연결합니다. 이는 성능을 개선하고 시스템을 단순화합니다.

Sandbox: 플러그 가능한 백엔드

샌드박스 실행 환경이 플러그 가능한 아키텍처로 개선되어, OpenShell 및 SSH 백엔드를 사용할 수 있습니다. 다양한 배포 환경에서 유연한 격리 환경을 구성할 수 있습니다.

5. 에이전트 엔진 혁신

OpenClaw 3.22의 에이전트 엔진은 장시간 대화 처리, 타임아웃 관리, 그리고 새로운 명령어로 사용자 경험을 크게 향상시켰습니다.

에이전트 엔진 개선사항

기능 이전 업그레이드
장문 대화 압축 수동 관리 필요 자동 연장 및 orphaned tool_result 자동 수정
기본 타임아웃 600초 (10분) 48시간
추가 기능 없음 /btw 명령어로 부가 질문 가능
장시간 대화 자동 압축

에이전트가 장시간 동작할 때 대화 히스토리가 너무 길어지는 문제를 자동으로 해결합니다:

  • 자동 연장: 데드라인이 임박하면 자동으로 연장
  • 자동 수정: orphaned tool_result 블록 (응답 없는 도구 호출 결과)을 자동으로 정리
  • 컨텍스트 최적화: 장시간 작업의 효율성 향상
타임아웃 대폭 연장

기본 에이전트 타임아웃이 600초(10분)에서 48시간으로 대폭 확대되었습니다. 이는 복잡한 다단계 작업, 대규모 데이터 처리, 대기 시간이 필요한 작업들을 완료할 수 있는 충분한 시간을 제공합니다.

/btw 명령어: 부가 질문 기능

새로운 /btw (By The Way) 명령어는 현재 진행 중인 작업의 컨텍스트에 영향을 주지 않으면서 측면 질문을 할 수 있게 해줍니다. 예를 들어, 대규모 작업을 진행 중일 때 "btw 이 부분이 왜 이렇게 설계되었나?"와 같은 질문을 던질 수 있으며, 에이전트는 메인 작업을 계속 추적하면서 질문에 답할 수 있습니다.
